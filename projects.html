Android App with Kotlin: Campus Dating

Github Repo

I collaborated with a team to develop an Android dating application for Waterloo students using Kotlin JetPack and Git workflows.

This project is developed to help UW-specific students meet/date people based on their preferences.

My contributions included designing and integrating multiple user interfaces to enhance the overall user experience. Additionally, we utilized Google Firebase to store user data and implemented an efficient algorithm to match users based on their preferences.

Waterloo Special Monopoly: "Watopoly"

Github Repo

I worked on the Monopoly Coding Project with another teammate, where we created a simple modified Monopoly game using C++. The game is specific to the University of Waterloo Campus, and there are added rules as well. The specification of this project can be found here.

We efficiently utilized Object-Oriented Programming (OOP) features in C++, implementing polymorphism, and also employed encapsulation by declaring critical game data as private within their respective classes.

The detailed design logic can be found here.

Walkthrough:

Command Interpretation:

  • roll (if the player can roll): The player rolls two dice, moves the sum, and takes action on the new square.
  • next (if the player cannot roll): Give control to the next player.
  • trade "name" "give" "receive": Offers a trade to "name" with the current player offering "give" and requesting "receive".
  • "property" buy/sell: Attempts to buy or sell an improvement for a property.
  • mortgage "property": Attempts to mortgage the property.
  • unmortgage "property": Attempts to unmortgage the property.
  • bankrupt: Declare bankruptcy (only if a player must pay more money than they have).
  • assets: Displays the assets of the current player.
  • all: Displays the assets of every player.
